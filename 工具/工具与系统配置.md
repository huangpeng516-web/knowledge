## 图像理解（MiniMax 直调 API）

**问题**：uvx MCP 依赖 PyPI 下载，大陆服务器访问受阻，图像理解一直超时挂起。

**解决方案**：绕过 MCP，直接调 MiniMax VLM API。

```bash
# 脚本路径
/root/.openclaw/workspace/scripts/understand_image_direct.py

# 调用方式
python3 /root/.openclaw/workspace/scripts/understand_image_direct.py <图片路径> "<prompt>"

# 关键参数
# - API Host: https://api.minimaxi.com
# - Endpoint: /v1/coding_plan/vlm
# - Model: MiniMax-Image-01 (在 payload 中传)
# - 图片格式: data:image/jpeg;base64,<base64数据>
```

**API Key 配置**：`~/.openclaw/config/minimax.json`

## OpenClaw 定时任务投递机制

**问题**：cron job 配置 `delivery.announce` 模式在 OpenClaw 3.24/3.28 均投递失败（not-delivered）。

**根因**：announce 投递机制与 isolated session 配合有 regression。

**解决方案**：
1. 去掉 cron job 的 `delivery` 配置
2. 在 payload 的 agent message 中直接嵌入 `message 工具调用指令`
3. 由子代理自己执行 message 工具发送

```python
# jobs.json 结构（正确示范）
{
  "payload": {
    "kind": "agentTurn",
    "message": "任务描述...\n完成后，用 message 工具发送到 openclaw-weixin，to=xxx@im.wechat，accountId=xxx，content=内容"
  }
}
```

**双通道发送**：微信 + QQ 同时发（在 message 中连续调用两次 message 工具）
