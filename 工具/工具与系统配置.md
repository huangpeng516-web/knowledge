## OpenClaw 定时任务投递机制

**问题**：cron job 配置 `delivery.announce` 模式在 OpenClaw 3.24/3.28 均投递失败（not-delivered）。

**根因**：announce 投递机制与 isolated session 配合有 regression。

**解决方案**：
1. 去掉 cron job 的 `delivery` 配置
2. 在 payload 的 agent message 中直接嵌入 `message 工具调用指令`
3. 由子代理自己执行 message 工具发送

```python
# jobs.json 结构（正确示范）
{
  "payload": {
    "kind": "agentTurn",
    "message": "任务描述...\n完成后，用 message 工具发送到 openclaw-weixin，to=xxx@im.wechat，accountId=xxx，content=内容"
  }
}
```

**双通道发送**：微信 + QQ 同时发（在 message 中连续调用两次 message 工具）

## 2026-04-05 重大异常记录

### 三项定时任务持续失败（根因：jobs.json被篡改后遗症）
截至2026-04-05，三项cron任务持续异常，问题相互关联：

| 任务 | 问题 | 根因 |
|------|------|------|
| 每日早间新闻推送 | announce模式cron状态ok，但lastDeliveryStatus=not-delivered，实际未送达 | announce投递机制与isolated session配合有regression |
| 每日日语N5短文 | 执行失败 | isolated session无法获取WeChat target参数 |
| 每日书籍章节总结 | 执行失败 | 同上 |

**jobs.json被篡改后遗症**：
- delivery字段被修改为`{"mode": "announce"}`，丢失了`channel`参数
- 导致QQ推送"假跑"三天（token仅56k，系统报成功但实际失败）
- jobs.json已修复，但announce模式本身的regression问题仍未解决

**announce模式"假成功"问题**：
- cron日志报'ok'不代表真实送达
- lastDeliveryStatus=not-delivered，但任务状态仍为completed
- 唯一解决方案：不用announce投递，改用子代理直接调用message工具

**手动补救有效**：
- 早间新闻于2026-04-05 00:14 UTC手动触发子代理，成功送达
- 说明子代理直接调用message工具是可靠的投递方式

### 新加坡VPS + OpenClaw 方案（第二阶段）

**用途**：解决 OpenClaw AI 访问外网问题，与 Netflix 解锁共用节点

**架构**：
```
OpenClaw 服务器（腾讯云境内）
    ↓ 设置代理环境变量
腾讯云新加坡VPS（mihomo 服务端，监听 7890）
    ↓
全球互联网（Google / Tavily / Netflix 等）

iOS 设备（Shadowrocket）
    ↓ 分流规则
腾讯云新加坡VPS（作为代理节点）
```

**VPS购买**：
- 腾讯云轻量应用服务器
- 地域：新加坡
- 系统：Ubuntu 22.04 LTS
- 规格：1核2G
- 带宽：5Mbps
- 费用：约¥30-50/月

**安全组放行**：TCP 22 / 7890 / 9090

**mihomo 服务端配置**（新加坡VPS）：
```yaml
port: 7890
socks-port: 7891
allow-lan: true
mode: rule
rules:
  - DOMAIN-SUFFIX,netflix.com,PROXY
  - DOMAIN-SUFFIX,google.com,PROXY
  - DOMAIN-SUFFIX,tavily.com,PROXY
  - GEOIP,CN,DIRECT
  - MATCH,DIRECT
```

**境内服务器 mihomo 客户端配置**：
```yaml
port: 7890
proxies:
  - name: sg-vps
    type: http
    server: YOUR_SG_IP
    port: 7890
```

**OpenClaw 环境变量**（境内服务器 /etc/environment）：
```
HTTPS_PROXY=http://127.0.0.1:7890
HTTP_PROXY=http://127.0.0.1:7890
ALL_PROXY=socks5://127.0.0.1:7891
```

**分流逻辑**：
- GEOIP,CN → 直连（国内网站）
- 其他国际网站（Google/Netflix/Tavily）→ 走新加坡 VPS

### iOS Shadowrocket 配置

- 添加新加坡 VPS 节点：类型 HTTP，服务器填VPS IP，端口 7890
- 全局路由：配置模式
- 分流规则：GEOIP CN → DIRECT；netflix.com/google.com → PROXY
